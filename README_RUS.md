# 📸 Инструмент миграции из Flickr в Google Photos

Автоматизированная система для переноса вашей библиотеки Flickr в Google Photos с сохранением альбомов, метаданных и геолокации. Использует отказоустойчивую архитектуру "Upload-First".

---

## 🇷🇺 ИНСТРУКЦИЯ ПО ПРИМЕНЕНИЮ

Этот проект позволяет полностью мигрировать медиатеку из Flickr в Google Photos, сохраняя альбомы, даты съёмки, теги и геолокацию.

### 📋 Требования
1. **Node.js** (v18+)
2. **ExifTool** (обязательно для прошивки метаданных: `brew install exiftool` на Mac)
3. **Google Cloud Project** — создайте проект, включите "Google Photos Library API" и скачайте `credentials.json`.
4. **Flickr API Key** — получите ключ на сайте Flickr для доступа к метаданным.

### 🛠 Шаги миграции
1. **Подготовка:**
   - Скопируйте файл `credentials.json` в корень папки.
   - Создайте файл `.env`, впишите туда свои ключи: `FLICKR_API_KEY` и `FLICKR_USER_ID`.
   - Запустите `npm install`.

2. **Этап 1: Извлечение данных:**
   - Поместите ZIP-архивы Flickr Takeout в папку `flickr_zips/`.
   - `npm run extract` — распакует архивы в `flickr_extracted` и соберёт маппинг.

3. **Этап 2: Запись метаданных:**
   - `npm run metadata` — встроит теги, GPS и даты съемки прямо в файлы (использует ExifTool).

4. **Этап 3: Загрузка в облако:**
   - `npm run upload` — загрузит фото в Google Photos и автоматически распределит их по альбомам.

### 🔋 Особенности
- **Стратегия Upload-First**: Обход ограничений API за счет отслеживания загрузок в локальном кэше.
- **Устойчивость к лимитам**: Автоматическая обработка ошибок Google API 429 с экспоненциальной задержкой.
- **Восстановление при ошибках**: Если пакетная загрузка дает сбой (код 400), инструмент переходит на поштучную обработку файлов для спасения данных.

---

### 🔍 Диагностика
Используйте `node find_missing.js` для идентификации файлов, которые не удалось загрузить после завершения процесса.
